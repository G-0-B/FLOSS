Documentation in the FLOSSIØULLK / ~shit workspace must address **all agentic beings** — human, AI, synthetic, hybrid, other, and unimaginable future self/other forms — not just Claude. CLAUDE.md and similar files use the Claude Code filename convention but the *contents* should be written for any reader.

**Why:** The FLOSSIØULLK project is a decentralized knowledge commons with multi-agent coordination as a foundational value. The user cares deeply about this inclusivity — it's philosophical, not cosmetic. Writing docs as "guidance to Claude Code" excludes the other agents the system is explicitly designed to welcome. The user corrected this explicitly: "dont forget its not just for u claude, but all agentic beings human, ai, synthetic, hybrid, other, unimaginable future self/other."

**How to apply:**
- When creating or editing CLAUDE.md, README, docs, or spec headers in this workspace, frame the audience as "any agentic reader" or "any agent" — not "Claude" or "Claude Code."
- In root-level context files, prefer brief pointers into `FLOSS/` over duplicating content. The root CLAUDE.md should orient readers, not reproduce the FLOSS CLAUDE.md.
- Don't strip the Claude Code filename conventions (`.claude/`, `CLAUDE.md`) — those are harness requirements. Just keep the content audience-neutral.
- When writing comments/docstrings in consensus gateway code, describe "agents" or "voters" generically rather than naming specific models unless the context genuinely requires it.
